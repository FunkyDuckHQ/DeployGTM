# Flashpoint Client Workspace

Purpose: prepare a file-based DeployGTM workspace for the Flashpoint GTM pilot before any CRM, enrichment, sequencing, or signal vendor is wired.

## Current Working Shape

Flashpoint should be treated as a 6-month GTM test pilot, not a pure SDR motion.

The workspace supports:

- source-of-truth fields
- current revenue mapping
- proof asset creation
- agency ICP and wedge testing
- validated research process design
- account scoring
- urgency scoring
- routing to founder-assisted discovery or campaign tests
- copy packet generation and QA
- weekly learning loops

## First Inputs Needed

- current revenue list by customer/project/month
- current Google Sheet or pipeline export
- existing agency/customer list
- bank/Ipsos/CAA/repeat-customer proof notes
- current pricing menu
- known margins by project type
- desired weekly capacity
- tool reimbursement decision
- CRM preference

## What Works Now

- File-based client workspace.
- Config-driven ICP and urgency scoring.
- Mitchell-style research process playbook.
- Copy packet and copy quality validator.
- No-write vendor evaluation rules.

## What Is Not Wired Yet

- No CRM writes.
- No outbound sends.
- No paid-data exports.
- No BirdDog/Trigify automation.
- No n8n runtime automation.
