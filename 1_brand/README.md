# 1 Brand

This folder turns research into positioning, voice, and message-market fit.

## Job

Build a brand and message system from customer language, buyer psychology, proof, and segment-specific angles.

## Inputs

- persona cards from `0_research/personas/`
- customer quotes from calls, interviews, and support conversations
- competitor notes
- sales objections
- proof points and case studies
- ICP and signal findings

## Core Outputs

- `voice-traits/`: voice trait cards sourced from actual customer language
- `value-props/`: value proposition cards by persona and segment
- `creative-angles/`: creative angle matrix by hook and audience
- `message-matrices/`: reusable outbound, ad, and call messaging matrices
- `brand-principles.md`: checklist for publishable copy

## Operating Rules

- Customer language is the brand voice.
- Prove every claim or mark it for review.
- Keep persona-specific angles separate.
- Use plain language.
- Do not let Octave, Claude, or any content tool become the system of record.
- Content tools implement the Content Adapter; DeployGTM owns the objects.

## First Pass Checklist

1. Extract quotes by theme.
2. Derive 3 to 4 voice traits.
3. Build value props by persona.
4. Build the creative angle matrix.
5. Create message matrices for the first test motion.
6. Mark blocked claims and proof requirements.
