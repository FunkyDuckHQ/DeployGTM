# Client Workflows

## Delivery Thesis

DeployGTM should be broad in promise, modular in delivery, opinionated in method, and flexible in scope.

The repeatable client motion is:

```text
Diagnose -> Design -> Activate -> Iterate
```

## Phase 1: Diagnose

Goal: understand the current GTM motion before building anything.

Inputs:

- business model
- current GTM motion
- CRM and tool stack
- data sources
- existing messaging
- ICP assumptions
- current reports
- sales process
- known gaps
- desired outcomes

Artifacts:

- current-state map
- systems map
- data/source inventory
- ICP assumption list
- workflow gap list
- build-vs-buy recommendations
- prioritized roadmap

## Phase 2: Design

Goal: decide what should be built, connected, simplified, or tested.

Artifacts:

- ICP/account prioritization logic
- signal map
- CRM field plan
- enrichment plan
- outbound testing plan
- playbook outline
- reporting model
- human approval rules

## Phase 3: Activate

Goal: turn the plan into a working operating layer.

Modules:

- account prioritization
- signal mapping
- CRM field audit
- enrichment workflows
- outbound experiment tracker
- reply classification
- playbook generator
- account research/profile generator
- client artifact generator
- weekly report generator
- vendor/tool stack recommendations

## Phase 4: Iterate

Goal: improve the motion based on market response and system evidence.

Iteration inputs:

- positive replies
- negative replies
- neutral or curious replies
- meetings booked
- account score movement
- signal quality
- CRM action completion
- seller feedback
- new client knowledge

Iteration outputs:

- updated assumptions
- updated ICP weights
- updated signal definitions
- new message tests
- new account routes
- playbook changes
- next weekly priorities

## Engagement Models

### Diagnose

Audit, current-state map, prioritized roadmap, and first-pass module recommendations.

### Build

Scoring, workflows, CRM actions, outbound tests, playbooks, alerts, and client artifacts.

### Partner

Ongoing iteration, reporting, workflow refinement, vendor support, and GTM advisory.

## What Not To Promise

Avoid retainers that are only "X meetings per month." Meetings matter, but the stronger promise is a learning and execution loop that improves account selection, messaging, routing, and sales action over time.
