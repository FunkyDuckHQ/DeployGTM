# Copy QA Checklist

Run this after draft generation. The QA pass must be separate from the drafting pass.

## Entity QA

- [ ] Company is correct.
- [ ] Person is correct.
- [ ] Domain or account ID confirms the entity.
- [ ] Duplicate-name risk has been addressed.
- [ ] Octave was not used as the first source of truth for identity.

## Context QA

- [ ] User-provided notes were considered first.
- [ ] Canonical repo/client workspace context was loaded.
- [ ] Drive or working brief context was loaded when available.
- [ ] Octave only enriched/interpreted after entity confirmation.
- [ ] Every specific claim traces to a source.

## Strategy QA

- [ ] Account/person brief exists before copy.
- [ ] Message strategy exists before copy.
- [ ] Pain is specific.
- [ ] Timing angle is supported.
- [ ] CTA fits the situation.
- [ ] Claims allowed and claims blocked are explicit.

## Copy QA

- [ ] Copy is not generic.
- [ ] Copy avoids fake certainty.
- [ ] Subject line is usable.
- [ ] CTA feels natural.
- [ ] It does not awkwardly mention DeployGTM.
- [ ] It sounds like Matthew: direct, specific, practical, not breathless.
- [ ] It avoids over-indexing on CRM unless CRM pain is supported.

## Banned Phrase Check

Reject or rewrite if the copy includes:

- `at DeployGTM`
- `I hope this finds you well`
- `just checking in`
- `quick question`
- `revolutionize`
- `game-changing`
- `cutting-edge`
- `AI-powered GTM engine`
- `signal layer` without explanation
- `unlock growth`
- `supercharge your pipeline`
- `we help companies like yours`
- `I noticed` without a source-backed observation

## Final Decision

- [ ] Pass
- [ ] Rewrite required
- [ ] Human review required
