# 0 Research

This folder is where a client engagement becomes grounded in data instead of opinions.

## Job

Find the real ICP, the real buying moments, and the real customer language before building campaigns.

## Inputs

- customer exports
- bookings, transactions, contracts, or revenue data
- CRM account and contact data
- sales call transcripts
- support calls
- customer interviews
- client-provided docs in Drive
- web research
- BirdDog or Clay-style signal findings

## Core Outputs

- `icp-definitions/`: one ICP definition per segment
- `personas/`: JTBD and Four Forces persona cards
- `research-processes/`: validated research processes with patterns, kill lists, and extraction rules
- `signal-definitions/`: client-specific alpha signals to monitor
- `account-lists/`: scored and sourced account lists
- `score-snapshots/`: ICP, urgency, engagement, and contact scores

## Operating Rules

- Start from existing customer economics when available.
- Segment by value and profitability, not only volume.
- Preserve actual customer language.
- Keep evidence and source refs attached to every important claim.
- Treat generic buying signals as weak until they become client-specific.
- Use research processes that can be tested against known examples.

## First Pass Checklist

1. Load the client working brief.
2. Identify existing customer data and gaps.
3. Build the first ICP hypothesis.
4. Define 3 to 5 alpha signal hypotheses.
5. Create one validated research process per signal family.
6. Score a small target account sample.
7. Write what changed in the client brief or handoff.
