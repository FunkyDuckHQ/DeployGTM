# DeployGTM — HubSpot Setup Playbook

*One-time configuration to connect the pipeline to HubSpot CRM. Run once per account.*

---

## Prerequisites

Before starting:
- [ ] HubSpot account exists (free tier works for contacts + sequences)
- [ ] `HUBSPOT_ACCESS_TOKEN` in `.env`
- [ ] `from_email` that will be used for sending (must be connected to HubSpot)
- [ ] Python dependencies installed (`make install`)

---

## Step 1: Get your HubSpot Access Token

1. Go to **HubSpot → Settings → Integrations → Private Apps**
2. Click **Create a private app**
3. Name it "DeployGTM Pipeline"
4. Under **Scopes**, enable:
   - `crm.objects.contacts.read` / `write`
   - `crm.objects.companies.read` / `write`
   - `crm.objects.deals.read` / `write`
   - `crm.schemas.contacts.read` / `write`
   - `automation` (for sequences)
   - `sales-email-read`
5. Click **Create app** → copy the access token
6. Add to `.env`:
   ```
   HUBSPOT_ACCESS_TOKEN=pat-na1-...
   ```

---

## Step 2: Create custom contact properties

Run the one-time setup command:
```bash
make setup-hubspot
```

This creates 10 custom properties on all HubSpot contacts:

| Property | Type | Purpose |
|----------|------|---------|
| Signal Source | Text | Where signal was detected (BirdDog, LinkedIn, etc.) |
| Signal Type | Dropdown | funding / hiring / gtm_struggle / agency_churn / tool_adoption |
| Signal Date | Date | When the signal fired |
| Signal Summary | Textarea | Free-text signal description |
| ICP Fit Score | Number | 1–5 (5 = perfect fit) |
| Signal Strength | Number | 1–3 (3 = active signal < 30 days) |
| Priority Score | Number | ICP Fit × Signal Strength (max 15) |
| Pain Hypothesis | Textarea | Claude-generated pain description |
| Enrichment Confidence | Dropdown | high / medium / low |
| Outreach Angle | Dropdown | founder_seller / first_sales_leader / revops_growth |

To verify: **HubSpot → Contacts → any contact → Property settings** — search "deploygtm".

## Step 3: Set up the deal pipeline

1. Go to **HubSpot → CRM → Deals → Pipeline settings**
2. Create a pipeline called "DeployGTM"
3. Add these stages in order:

| Stage | Win probability |
|-------|----------------|
| Signal Detected | 5% |
| Outreach Sent | 10% |
| Reply Received | 25% |
| Discovery Call | 40% |
| Proposal Sent | 60% |
| Closed Won | 100% |
| Closed Lost | 0% |
| Nurture / Waiting | 10% |

## Step 4: Connect your sending email

Sequences require a connected inbox:

1. **HubSpot → Settings → General → Email**
2. Click **Connect personal email**
3. Connect the Gmail or Outlook account you send outreach from
4. Verify it works by sending a test

This is the `from_email` address you'll use in config.yaml.

## Step 5: Create sequences for each persona

Create one sequence per buyer persona. Each sequence is a 3-touch email cadence.

### How to create a sequence:
1. **HubSpot → Automation → Sequences**
2. Click **Create sequence**
3. Name it exactly as shown below
4. Add email steps (HubSpot calls them "enrollment steps")

## Step 6: Get the sequence IDs

After creating each sequence:
1. Open the sequence in HubSpot
2. Look at the URL: `https://app.hubspot.com/sequences/<portal-id>/edit/<SEQUENCE_ID>`
3. Copy the numeric ID at the end

---

## Step 7: Update config.yaml

```yaml
tools:
  hubspot:
    enabled: true
    require_confirmation: true
    sandbox: false
    sequences:
      founder_seller: "12345678"        # ← paste sequence ID here
      first_sales_leader: "23456789"    # ← paste sequence ID here
      revops_growth: "34567890"         # ← paste sequence ID here
    from_email: "matt@yourcompany.com"  # ← the connected inbox
```

---

## Step 8: Test the full setup

**Dry run (no data written):**
```bash
python scripts/hubspot.py setup-properties --dry-run
python scripts/hubspot.py push --file output/<any_file>.json --dry-run
python scripts/hubspot.py enroll --file output/<any_file>.json --dry-run
```

**Live run (writes to HubSpot):**
```bash
python scripts/hubspot.py push --file output/<any_file>.json
```

HubSpot requires confirmation (`require_confirmation: true`) before writing — you'll see a prompt.

---

## Step 9: Verify in HubSpot

After a live push:
1. **HubSpot → Contacts** — search for the contact you just pushed
2. Check that all custom properties have values
3. **HubSpot → Companies** — verify the company record was created/updated
4. **HubSpot → Automation → Sequences** — verify enrollment shows the contact

---

## Troubleshooting

**`401 Unauthorized`:** Access token is wrong or expired. Re-generate in HubSpot → Private Apps.

**`Property not found`:** Custom properties weren't created. Run `make setup-hubspot`.

**`Sequence enrollment failed`:** Common causes:
- Sequence ID is wrong — re-check URL
- `from_email` is not connected to HubSpot — re-connect in Settings → Email
- Contact already enrolled in this sequence — HubSpot blocks duplicate enrollments

**`Contact/company upsert failed`:** Check that `email` and `domain` fields are populated in the pipeline output. The upsert uses email as the contact key and domain as the company key.

---

## Ongoing maintenance

- Run `make setup-hubspot` again if you add new custom properties
- Update sequence IDs in config.yaml if you rebuild sequences
- Add new persona sequences by adding a key to `tools.hubspot.sequences` in config.yaml and `detect_persona()` in outreach.py
